Note

If you set parameters differently from default, you need to ensure they stay compatible across workflow steps. For example, when creating a dataset with --data_type mesh and/or --feature_type rgb, training on that dataset will only work if you also set these parameters for training.

Note

For detailed parameter definitions and their complete applicability, refer to the args.py file. Each workflow step has its own parameter class (e.g., TrainingAppArgs for training) that inherits from parent classes.
